What one command does

Run uxinspect once. Get a report you can hand to anyone on the team.

1

Loads in a real browser

Headless Chromium, Firefox, or WebKit. Real network. Real rendering. Real DOM.

2

Discovers every interactive element

Buttons, links, forms, menus, modals. Walks the DOM, finds them all.

3

Clicks them all, every viewport

Mobile, tablet, desktop. Every button, every screen, every state.

4

Records every step (Pro)

rrweb captures DOM events to .uxinspect/replays/. Local. No cloud.

5

Attributes errors to the click (Pro)

Console errors and 4xx/5xx network calls pinned to the exact step.

6

Hands you a report

report.html with screenshots, scores, and a "Replay this failure" link on every red row.

Pro turns a green report into a debuggable failure

Free MIT runs every audit. Pro adds the parts that let you find why a click broke and reproduce it.

Replay

Replay capture (rrweb local)

Every flow + every explore run is recorded to .uxinspect/replays/. Bundled rrweb-player, no CDN.

Replay

"Replay this failure" link

Failed flows in report.html link to the viewer at the failure timestamp. Click and see the broken click.

Assertions

Per-step assertion DSL

assert: { console: 'clean', network: 'no-4xx', dom: 'no-error', visual: 'matches' } on every step.

Attribution

Per-click console + network

Capture resets before every step. The 500 / TypeError is attributed to the click that fired it.

Verifiers

Stuck-spinner detection

If [aria-busy] or .spinner persists past 5s after a click, flagged broken.

Verifiers

Disabled-button audit

Walks every [disabled], attempts click, asserts no state change. Catches the bugs you forgot.

Verifiers

DOM error-state appearance

Scans for new [role="alert"], error toasts, .alert-danger after every click.

Coverage

Click coverage % per route

Interactive-element count vs clicked count. --coverage-min 80 budget flag for CI.

Coverage

Auth-gated route walker

storageState + auto-discovered protected routes. Per-click verifier on every gated page.

History

SQLite history + trends

better-sqlite3 in .uxinspect/history.db. Z-score anomaly detection on every metric.

Visual

Cross-browser matrix UI

Side-by-side Chromium / Firefox / WebKit screenshots in the report with diff overlay toggle.

AI

Locator caching + self-healing

Cache resolved locators by selector hash. Retry neighboring strategies on miss. 2x faster reruns.

Drop a config file, run the CLI, get HTML + JSON + JUnit + SARIF + scrubbable replays.

// uxinspect.config.ts
export default {
  url: 'https://yourapp.com',
  viewports: [
    { name: 'desktop', width: 1280, height: 800 },
    { name: 'mobile',  width: 375,  height: 667 },
  ],
  flows: [{
    name: 'signup',
    steps: [
      { goto: 'https://yourapp.com/signup' },
      { fill: { selector: '#email', text: 'me@example.com' } },
      { click: 'button[type=submit]',
        assert: { console: 'clean', network: 'no-4xx', dom: 'no-error' } },
      { waitFor: '.welcome' },
    ],
  }],
  checks: { a11y: true, perf: true, visual: true, explore: true, replay: true },
  reporters: ['html', 'json', 'junit', 'sarif'],
};
